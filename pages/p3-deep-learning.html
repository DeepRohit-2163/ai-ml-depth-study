Backpropagation & Chain Rule

Intuition First

Backprop is the algorithm for computing how much each weight contributed to the loss — so we can nudge each weight in the direction that reduces loss. It's just the chain rule of calculus, applied efficiently via dynamic programming (computing gradients from output back to input).

Forward pass: Compute predictions layer by layer: z¹ = W¹x + b¹, a¹ = σ(z¹), z² = W²a¹ + b², etc. Store intermediate values (needed for backward pass).

Backward pass (chain rule): Compute ∂L/∂W at every layer by applying chain rule from output to input:

∂L/∂W¹ = ∂L/∂a² · ∂a²/∂z² · ∂z²/∂a¹ · ∂a¹/∂z¹ · ∂z¹/∂W¹

Computational graph: DAG of operations. Forward pass builds the graph; backward pass traverses it in reverse, accumulating gradients via the chain rule. PyTorch autograd does this automatically by tracking operations on tensors with requires_grad=True.

Vanishing gradients: When gradients are multiplied through many layers, if each factor < 1 (e.g., sigmoid derivative max = 0.25), the product → 0. Early layers receive near-zero gradient → don't learn. Solutions: ReLU (derivative = 1 for positive inputs), residual connections (gradient flows around layers), BatchNorm (re-scales activations), proper weight initialization (Xavier/He).

Exploding gradients: Product of factors > 1 → gradient → ∞. Fix: gradient clipping (clip gradient norm to a threshold, e.g., 1.0). Common in RNNs.

Weight initialization: If initialized too small → vanishing. Too large → exploding. Xavier: std = √(2/(n_in + n_out)) — for tanh/sigmoid. He: std = √(2/n_in) — for ReLU. Goal: keep activation variance ~constant across layers.

The Interrogation

Mid Trace the gradient through a single sigmoid neuron from loss to weight. ▼

Setup: z = wᵀx + b, a = σ(z), L = (a - y)²/2.

∂L/∂a = (a - y) ∂a/∂z = σ(z)·(1-σ(z)) [sigmoid derivative] ∂z/∂w = x ∂L/∂w = ∂L/∂a · ∂a/∂z · ∂z/∂w = (a-y)·σ(z)(1-σ(z))·x

Note: σ(z)·(1-σ(z)) ≤ 0.25. With 10 sigmoid layers: 0.25^10 ≈ 10^-6. This is the vanishing gradient problem. With ReLU: ∂a/∂z = 1 (for z > 0) → gradient flows through unattenuated.

Depth Probe Why do residual connections help with vanishing gradients? ▼

Residual connection: output = F(x) + x (identity shortcut). Gradient through this block:

∂L/∂x = ∂L/∂output · (∂F/∂x + 1)

The "+1" ensures gradient always has a direct path: even if ∂F/∂x → 0 (vanishing), ∂L/∂x ≥ ∂L/∂output. The identity shortcut is a "gradient highway" that bypasses the transformation. This is why ResNets can train with 100+ layers without vanishing gradient problems. The same principle operates in Transformers: every sub-layer (attention, FFN) has a residual connection.

Optimizers: SGD → Adam → AdamW

Amazon Quote (2025)

"Don't just say 'I used AdamW.' Explain why — specifically, weight decay decoupling." This is a direct quote from an Amazon Applied Scientist interview. Know what AdamW does differently from Adam.

SGD with momentum: v ← β·v - α·∇L; θ ← θ + v. Momentum accumulates velocity in the direction of consistent gradients → faster convergence in shallow dimensions, dampens oscillations. Nesterov: compute gradient at look-ahead position → slightly better theoretical convergence.

RMSprop: Adapts learning rate per-parameter. Maintains v = β·v + (1-β)·(∇L)². Update: θ ← θ - α·∇L/√(v+ε). Parameters with large gradients get smaller effective learning rate → prevents explosive updates. ε for numerical stability.

Adam = Momentum + RMSprop. First moment (m) = exponential moving avg of gradient. Second moment (v) = exponential moving avg of squared gradient. Bias correction: m̂ = m/(1-β₁ᵗ), v̂ = v/(1-β₂ᵗ) (corrects for initialization bias at t=0). Update: θ ← θ - α·m̂/√(v̂+ε). Default: β₁=0.9, β₂=0.999, ε=1e-8.

m_t = β₁·m_{t-1} + (1-β₁)·g_t v_t = β₂·v_{t-1} + (1-β₂)·g_t² θ_t = θ_{t-1} - α·m̂_t / (√v̂_t + ε)

Adam with L2 (naive weight decay): Loss = L + λ·‖θ‖². Gradient = ∇L + λ·θ. The L2 penalty gets included in m and v — it's scaled by the adaptive learning rate. This means features with large gradients get more weight decay than features with small gradients — inconsistent regularization.

AdamW (decoupled weight decay): Separate the weight decay from the gradient update: θ ← θ·(1-α·λ) - α·m̂/√(v̂+ε). Weight decay is applied directly to θ, not through the gradient — unaffected by m and v. This is correct weight decay. In Adam with L2, high-gradient parameters get less actual regularization because v̂ scales it down. AdamW is now the default for training Transformers.

The Interrogation

Killer What exactly is wrong with Adam + L2 regularization, and how does AdamW fix it? ▼

In Adam + L2: the regularization term λ·θ is added to the gradient → it goes into m_t and v_t. For parameters with large gradient history (large v_t), the adaptive learning rate α/√v_t is small. This small effective learning rate applies equally to both the task gradient AND the regularization gradient. So parameters that already have large gradients (important weights) get both small task updates AND small regularization updates. The regularization is inconsistent across parameters. In AdamW: weight decay is applied directly: θ ← θ - α·[m̂/√v̂] - α·λ·θ. The regularization step is separate from the adaptive gradient step. Every parameter gets exactly α·λ·θ weight decay regardless of its gradient history. This is theoretically correct decoupling and is why AdamW generalizes better for transformers.

BatchNorm vs LayerNorm

Amazon Asked This (2025)

"When does BatchNorm fail? Why do LLMs always use LayerNorm?" — Both questions appeared in Amazon's 2025 ML Deep Dive. Know the failure modes, not just the definitions.

Why normalize at all? Internal covariate shift: as weights update, the distribution of each layer's inputs changes, forcing later layers to constantly adapt. Normalization stabilizes these distributions → allows higher learning rates → faster training.

BatchNorm: Normalizes across the batch dimension. For a feature j: μ_j = mean over batch, σ_j = std over batch. Then: x̂ = (x - μ_j)/σ_j, output = γ·x̂ + β (learnable scale/shift). At inference: uses running mean/std accumulated during training.

BatchNorm: normalize over N (batch size) for each feature dimension

BatchNorm failure modes: (1) Small batch sizes: With batch=1, μ and σ are estimated from 1 sample — meaningless statistics. Gradient through these estimates is also noisy. (2) Sequence models / variable length: Different positions in a sequence shouldn't be normalized together. (3) Inference with different distribution: Running stats from training may not match test distribution (domain shift → bad normalization). (4) Recurrent networks: Each timestep would need different BatchNorm stats. (5) Multi-GPU training: Statistics must be synchronized across GPUs — expensive communication overhead.

LayerNorm: Normalizes across the feature dimension for each individual sample. For sample i: μ_i = mean over all features, σ_i = std over all features. No dependence on batch size, no running stats needed. Works identically at training and inference. Perfect for: Transformers (variable length sequences), RNNs, small or variable batch sizes.

LayerNorm: normalize over D (features/channels) for each sample

Why Transformers use LayerNorm (not BatchNorm): (1) NLP batch sizes can be small (memory constrained by long sequences). (2) Sequences are variable length — BatchNorm can't normalize across padding. (3) LayerNorm is computed independently per token — consistent across sequence positions. (4) No running statistics needed → simpler deployment.

The Interrogation

Depth Probe Explain BatchNorm at inference time — how is it different from training? ▼

At training: BatchNorm normalizes using the current mini-batch's μ and σ. Simultaneously, it maintains running averages: μ_run = α·μ_run + (1-α)·μ_batch. At inference: you typically have batch_size=1 — can't compute meaningful batch statistics. So BatchNorm uses the running averages from training (μ_run, σ_run) instead of the current batch. The network is set to eval mode (model.eval() in PyTorch), which switches from batch statistics to running statistics. Bug: if you forget to call model.eval(), BatchNorm normalizes using the single test sample's statistics (useless with batch=1) → poor inference performance. Second bug: if test distribution differs from training distribution (covariate shift), running stats are wrong → use test-time BatchNorm or LayerNorm instead.

Dropout & Regularization in NNs

Dropout: Randomly zero out each neuron with probability p during training. Forces the network to learn redundant representations — no single neuron can be relied upon. At test time: scale activations by (1-p) to maintain expected value (inverted dropout: scale by 1/(1-p) at train time → no scaling needed at test time — modern standard).

Ensemble interpretation: With n neurons and dropout probability p, you train 2^n different thinned networks simultaneously (one per subset of active neurons). At inference, using all neurons with scaled weights is an approximation of averaging all these networks (model averaging).

Bayesian interpretation: Dropout is equivalent to approximate variational inference in a Bayesian neural network. Each dropout mask corresponds to sampling from a distribution over network weights. Test-time dropout (MC Dropout) gives uncertainty estimates by averaging multiple stochastic forward passes.

Where to use dropout: After dense layers (typically p=0.5). Rarely in conv layers (p=0.2 if used). NOT inside attention layers in Transformers (attention dropout is separate). NOT between BatchNorm and its following layer (causes variance shift).

Other NN regularization: Weight decay (L2 on weights), early stopping (track validation loss, stop when it starts increasing), data augmentation (effectively increases dataset size), label smoothing (replace hard targets 0/1 with 0.1/0.9 → prevents overconfidence).

The Interrogation

Depth Probe Why does dropout need to scale activations at test time? ▼

During training with dropout p: each neuron is active with probability (1-p). Expected output of a neuron = (1-p)·actual output. At test time with no dropout: all neurons are active, so output = actual output > expected training output. This breaks the scale of the next layer's inputs. Fix 1 (classic dropout): at test time, multiply all activations by (1-p) to restore expected value. Fix 2 (inverted dropout, modern standard): at training time, divide active activations by (1-p) → expected output at training = actual output. At test time: no scaling needed. PyTorch uses inverted dropout — the model works correctly at test time without any scaling, making deployment simpler.

CNNs (conv, pooling, receptive field)

Convolution operation: Slide a kernel K of size k×k over input X, computing dot product at each position. Output(i,j) = Σ Σ K(m,n)·X(i+m, j+n). This is cross-correlation (not true convolution) but called conv in DL.

Output size formula: (W - K + 2P)/S + 1, where W=input width, K=kernel size, P=padding, S=stride. Internalize this: if W=32, K=3, P=1, S=1 → (32-3+2)/1+1 = 32. Same padding preserves spatial size.

Parameter sharing: Same kernel weights applied at every position → massive reduction in parameters vs fully connected. A conv layer with 64 filters of 3×3 = 64·3·3·in_channels parameters, regardless of input image size. Vs FC: would be 64·H·W·in_channels — grows with image size.

Receptive field: The region of the input a single output neuron "sees." For a 3×3 conv: receptive field = 3×3. Stacking L layers of 3×3 convs: receptive field = (2L+1)×(2L+1). Dilated conv with rate d: effectively k + (k-1)(d-1) per layer — increases receptive field without more parameters.

Pooling: Max pool: take max in each window → selects strongest feature, some translation invariance. Average pool: take mean → smoother. Global Average Pool: average entire feature map to a single value per channel — used before final classifier in modern CNNs (replaces massive FC layers). Stride-2 conv often replaces max pool in modern architectures.

Why CNNs for images: (1) Translation invariance — a cat detector works anywhere in the image. (2) Local connectivity — pixels near each other are related. (3) Parameter efficiency — shared weights. (4) Hierarchical features — early layers: edges; mid layers: textures; deep layers: objects.

The Interrogation

Mid Calculate the output size of: 64×64 input, 3×3 kernel, padding=0, stride=2. ▼
(W - K + 2P)/S + 1 = (64 - 3 + 0)/2 + 1 = 61/2 + 1 = 30.5 + 1 → floor = 31

Output: 31×31. The floor operation is applied when (W - K + 2P) is not divisible by S. With padding=1, stride=2: (64-3+2)/2+1 = 63/2+1 = 32.5 → floor = 32. Padding=1 with stride=2 halves the spatial dimensions cleanly.

RNNs, LSTMs & Vanishing Gradients

Vanilla RNN: h_t = tanh(W_h·h_{t-1} + W_x·x_t + b). The same weights W_h are applied at every timestep. Long-range dependencies: for gradient to flow from t=T to t=0, it must pass through T applications of W_h·diag(tanh'(·)).

Vanishing gradient in RNNs: If ‖W_h‖ < 1 or tanh' < 1, then the gradient product → 0 as T increases. The network can't learn that a word at position 1 influences the output at position 100. Exploding: if ‖W_h‖ > 1 → gradient → ∞ → NaN. Fix: gradient clipping.

LSTM (Long Short-Term Memory): Adds a cell state c_t (highway) and three gates:
• Forget gate: f_t = σ(W_f·[h_{t-1}, x_t]) — what fraction of c_{t-1} to keep
• Input gate: i_t = σ(W_i·[h_{t-1}, x_t]) — what new info to write to cell state
• Cell candidate: c̃_t = tanh(W_c·[h_{t-1}, x_t]) — new content to write
• Cell update: c_t = f_t ⊙ c_{t-1} + i_t ⊙ c̃_t
• Output gate: o_t = σ(W_o·[h_{t-1}, x_t]); h_t = o_t ⊙ tanh(c_t)

Why LSTM solves vanishing gradients: Gradient through the cell state path: ∂c_t/∂c_{t-1} = f_t. If forget gate ≈ 1 (sigmoid output near 1), gradient flows through unattenuated. This is a learned, adaptive gradient highway — the model can learn to keep relevant information indefinitely. The gates are trained to open/close as needed.

LSTM vs Transformer: LSTMs can't be parallelized (each timestep depends on previous hidden state). Transformers process all positions in parallel using attention. For most NLP tasks, Transformers have replaced RNNs. LSTMs still used for: streaming data, low-latency requirements, or very long sequences where attention is too expensive.

The Interrogation

Depth Probe Explain precisely why LSTM's cell state solves vanishing gradients that RNN's hidden state doesn't. ▼

RNN gradient path: ∂h_T/∂h_0 = Π_{t=1}^{T} W_h·diag(tanh'(·)). Each factor ≤ ‖W_h‖·max(tanh') = ‖W_h‖·1. If ‖W_h‖ < 1, product → 0 exponentially. LSTM cell state gradient path: ∂c_t/∂c_{t-1} = f_t (elementwise, no matrix multiply through a saturating nonlinearity). The forget gate f_t = σ(·) ∈ (0,1). If the network learns to set f_t ≈ 1 (keep everything), gradient ≈ 1 at each step → no vanishing. The key difference: the cell state has a direct, multiplicative skip connection through time (c_t = f_t⊙c_{t-1} + ...) unlike the hidden state which goes through tanh and W_h at every step. This is the same intuition as residual connections in CNNs — a direct path for gradients.

Activation Functions & Dead Neurons

Sigmoid / tanh: Saturate at extremes → near-zero gradient → vanishing gradients. Sigmoid also not zero-centered (outputs 0 to 1) → all gradients for weights are same sign → zig-zag gradient descent. Tanh is zero-centered but still saturates.

ReLU: max(0, x). Non-saturating for x>0 → gradient = 1 → no vanishing for positive inputs. Fast to compute. Problem: Dead ReLU — if a neuron's input is always negative, output is always 0, gradient is always 0 → weight never updates → neuron "dies." Causes: large learning rate (overshoot into dead zone), poor initialization (biases too negative). Fix: Leaky ReLU (max(0.01x, x)), ELU, or proper initialization.

GELU (Gaussian Error Linear Unit): x·Φ(x) where Φ is the Gaussian CDF. GELU(x) = x·0.5·(1+erf(x/√2)). Smooth approximation to ReLU. Stochastic interpretation: randomly zero out x with probability 1-Φ(x). Used in BERT, GPT, most modern Transformers. Why over ReLU: (1) Smooth (differentiable everywhere) → cleaner gradients. (2) Can have small negative outputs near 0 → richer representation. (3) Empirically better performance on NLP tasks.

SiLU / Swish: x·σ(x). Used in newer models (LLaMA 2/3 uses SwiGLU = SiLU gated linear unit in FFN). Similar properties to GELU.

The Interrogation

Depth Probe Why did GELU replace ReLU in Transformers — and what does it actually compute? ▼

ReLU has two problems for Transformers: (1) Hard threshold at 0 → non-smooth → gradient discontinuity at x=0. (2) Strictly non-negative output → can't represent negative information. GELU: x·Φ(x). At large negative x: Φ(x)→0 → output→0. At large positive x: Φ(x)→1 → output≈x. Near x=0: smooth transition. This is like a "soft ReLU" that stochastically gates activation — for each neuron with input x, GELU computes "let x pass through with probability Φ(x)." This means: small positive values are partially gated (not fully passed), small negative values are partially negative (not fully zeroed). This richer, smooth gating empirically helps Transformers learn more nuanced representations. The smoothness also makes optimization easier. BERT paper (Devlin 2018) used GELU; GPT-2 used GELU; it's become the default for Transformers unless replaced by SwiGLU.
