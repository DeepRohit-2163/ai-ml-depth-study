Linear Regression

Intuition First

Linear regression finds the best-fit hyperplane through data by minimizing the sum of squared errors. "Best fit" has an exact mathematical definition and a closed-form solution. Understanding when that solution works and when gradient descent is better is a common MNC depth probe.

Model: ŷ = Xθ (matrix form). Loss: MSE = (1/n)·‖y - Xθ‖²

Normal equation (closed form): Set ∇L = 0 → θ* = (XᵀX)⁻¹Xᵀy. Exact solution. Requires XᵀX to be invertible (features not perfectly collinear). O(n³) cost to invert — infeasible when n (features) is large.

Assumptions: Linearity, independence of errors, homoscedasticity (constant variance of errors), normality of errors (for inference, not prediction). When violated: predictions still work, but confidence intervals break.

R² (coefficient of determination): Fraction of variance explained by the model. R² = 1 - SS_res/SS_tot. R² = 1 → perfect fit. R² = 0 → model does no better than predicting the mean. Caution: R² always increases with more features (use adjusted R²).

Multicollinearity: When features are correlated, XᵀX becomes near-singular. Coefficient estimates become unstable (high variance). Fix: remove correlated features, PCA, or add L2 regularization (Ridge).

The Interrogation

Depth Probe Derive the normal equation from scratch. ▼

L(θ) = ‖y - Xθ‖² = (y - Xθ)ᵀ(y - Xθ). Expand:

L = yᵀy - 2θᵀXᵀy + θᵀXᵀXθ

Take gradient w.r.t. θ and set to zero:

∇L = -2Xᵀy + 2XᵀXθ = 0 → XᵀXθ = Xᵀy → θ* = (XᵀX)⁻¹Xᵀy

This is unique when XᵀX is invertible (features linearly independent). When not invertible (rank-deficient), use pseudoinverse or add regularization: θ* = (XᵀX + λI)⁻¹Xᵀy (Ridge regression). The λI term also makes the matrix always invertible.

Mid When does gradient descent beat the normal equation? ▼

Use GD when: (1) n (features) > ~10,000 because O(n³) inversion becomes too slow, (2) data doesn't fit in memory (use mini-batch SGD), (3) problem is non-linear (no closed form exists), (4) online learning — new data arrives continuously. Use normal equation when: n is small, you want the exact global optimum, and memory is not a constraint. In deep learning, there's never a closed form — always GD.

Logistic Regression & Loss Function

Amazon Asked This Directly (2025)

"Write the loss function for logistic regression and prove that it has a global minimum." This is a real question from Amazon's Applied Scientist interview. Know this cold.

Model: P(y=1|x) = σ(wᵀx + b) where σ(z) = 1/(1+e⁻ᶻ). Output is a probability (0-1).

Why sigmoid? We want outputs ∈ (0,1) that represent probabilities. Sigmoid is the canonical "squash to probability" function. It's also the CDF of the logistic distribution. Under the Bernoulli likelihood model, sigmoid emerges naturally.

Loss function (cross-entropy / log-loss): Derived from maximizing the Bernoulli likelihood:

L = -1/n · Σ [y_i·log(ŷ_i) + (1-y_i)·log(1-ŷ_i)]

Why not MSE for classification? MSE with sigmoid activations creates a non-convex loss surface with vanishing gradients near 0 and 1. Cross-entropy remains convex and gives larger gradients when predictions are confidently wrong.

Convexity (global minimum): The log-loss is a convex function of w. Proof: the Hessian ∇²L = XᵀDX where D = diag(ŷ_i(1-ŷ_i)) ≥ 0 (positive semi-definite). A PSD Hessian means the function is convex — any local minimum is the global minimum. GD will always find the global optimum (given sufficient iterations and learning rate).

Class imbalance handling: Class weights in loss: L = -Σ [w_pos·y·log(ŷ) + w_neg·(1-y)·log(1-ŷ)]. Or use oversampling (SMOTE), undersampling, or threshold tuning post-training.

The Interrogation

Killer Prove that logistic regression's log-loss has a global minimum (convexity proof). ▼

Loss: L(w) = -Σ [y_i·log σ(wᵀx_i) + (1-y_i)·log(1-σ(wᵀx_i))]. Compute gradient:

∇L = -Σ (y_i - σ(wᵀx_i)) · x_i = Xᵀ(σ(Xw) - y)

Compute Hessian:

∇²L = Xᵀ D X where D = diag(σ(wᵀx_i)·(1-σ(wᵀx_i)))

Since σ(z)∈(0,1), each diagonal entry of D > 0. Therefore D is positive definite, and ∇²L = XᵀDX is positive semi-definite (PSD). A PSD Hessian means L is convex. For a convex function, every local minimum is a global minimum. Therefore logistic regression's loss has a global minimum, and GD converges to it.

Mid Why can't you use MSE instead of cross-entropy for logistic regression? ▼

Two reasons: (1) Non-convexity: MSE + sigmoid creates a non-convex loss surface with many local minima — GD may not converge to the best solution. (2) Vanishing gradients: When the model is confidently wrong (ŷ ≈ 0 but y=1), the sigmoid saturates and its gradient → 0. MSE loss gradient = (ŷ-y)·σ'(z) → near 0. Learning stalls. Cross-entropy loss gradient = (ŷ-y) — no σ'(z) term — so gradients stay large when predictions are wrong. This is why cross-entropy is the correct loss for classification under MLE.

Decision Trees (Entropy, Gini, IG)

Amazon Asked This Directly (2025)

"Manually calculate Entropy, Gini Index, and Information Gain for dataset splits." This was a live calculation question in Amazon's ML Deep Dive round. Practice doing these by hand.

Splitting criterion goal: Find the split that best separates classes. "Best" = most purity = least entropy/Gini after the split.

Entropy: H(S) = -Σ p_i·log₂(p_i). Max (most impure) when all classes equally likely. Zero when perfectly pure.

Information Gain: IG(S,A) = H(S) - Σ_v (|S_v|/|S|)·H(S_v). How much entropy is reduced by splitting on attribute A. Choose the attribute with highest IG.

Gini Index: Gini(S) = 1 - Σ p_i². Ranges 0 (pure) to 0.5 (binary equally split). Computationally faster than entropy (no log). Used by default in sklearn's CART for Random Forest.

Bias of IG: Information Gain favors features with many distinct values (e.g., IDs). Fix: use Gain Ratio = IG(S,A) / H(A).

Stopping criteria: Max depth, min samples per leaf, min impurity decrease, or pure nodes. Pruning: Pre-prune (stop early) or post-prune (grow full tree, then remove nodes that don't improve validation performance).

The Interrogation

Depth Probe Manually compute Gini and Information Gain for a binary split. ▼

Root: 10 examples, 6 positive, 4 negative. Split A produces: Left={4+, 0-}, Right={2+, 4-}.

H(Root) = -(0.6·log₂0.6 + 0.4·log₂0.4) = 0.971 bits H(Left) = -(1·log₂1 + 0) = 0 bits (pure) H(Right) = -(0.333·log₂0.333 + 0.667·log₂0.667) = 0.918 bits IG = 0.971 - (4/10·0 + 6/10·0.918) = 0.971 - 0.551 = 0.42 bits
Gini(Root) = 1 - (0.6² + 0.4²) = 1 - 0.52 = 0.48 Gini(Left) = 0 (pure) Gini(Right) = 1 - (0.333² + 0.667²) = 0.444 Weighted Gini after split = (4/10·0 + 6/10·0.444) = 0.267

Lower weighted Gini = better split. Compare this value across all candidate splits to choose the best.

Mid Why do decision trees overfit and how do you fix it? ▼

An unbounded decision tree will grow until every leaf is pure — memorizing the training data (zero train error). This is extreme overfitting: the model has learned noise, not signal. Fixes: (1) Max depth: limit how deep the tree grows. (2) Min samples per leaf: don't split a node with fewer than k examples. (3) Pre-pruning: stop if IG of best split < threshold. (4) Post-pruning (cost-complexity): grow full tree, then remove subtrees that don't improve performance on a holdout set. (5) Use ensembles: Random Forest (bagging) or Gradient Boosting — the high variance of individual trees averages out.

Random Forests & Bagging

Bagging (Bootstrap Aggregating): Train k models, each on a bootstrap sample (sampling with replacement) of the training data. Average predictions (regression) or majority vote (classification). Reduces variance without increasing bias. Each model sees ~63.2% of original data (some points duplicated, ~36.8% never seen = OOB).

Random Forest = Bagging + Feature Subsampling. At each split, only consider a random subset of √n_features features. This decorrelates the trees — if one feature is very strong, plain bagging would make all trees similar (correlated). Decorrelated trees → bigger variance reduction when averaged.

OOB (Out-of-Bag) error: Each tree is trained on a bootstrap sample. The ~36.8% left-out samples are the OOB set for that tree. Average OOB error across all trees = free validation estimate without a separate validation set.

Feature importance: Average decrease in impurity (Gini/entropy) when a feature is used for a split, averaged across all trees. Or: permutation importance — permute a feature's values and measure degradation in OOB accuracy.

When RF fails: Extrapolation (RF can't predict outside the range of training values). Very noisy labels. When features have very different scales (trees are scale-invariant but RF with many features may have issues). High cardinality categorical features.

The Interrogation

Mid Why does feature subsampling in RF reduce variance more than plain bagging? ▼

In plain bagging, if one feature is highly predictive, nearly all trees will split on it at the top. The trees are correlated. Variance of average of correlated variables = ρσ² + (1-ρ)σ²/n, where ρ is the correlation. When ρ is high, averaging many correlated trees barely reduces variance. Feature subsampling forces each tree to consider different features at each split — trees diverge, reducing ρ. Lower ρ → better variance reduction from averaging. This is why RF often outperforms plain bagging despite each individual tree being slightly worse (it hasn't seen all features).

Depth Probe Bagging vs Boosting — when do you use each? ▼

Bagging: Trains models independently in parallel. Reduces variance. Best when base model is high-variance (e.g., deep decision trees). Works well when you have a complex but noisy dataset and risk of overfitting. RF is robust and hard to overfit with more trees. Boosting: Trains models sequentially, each correcting the previous one's errors. Reduces bias (and variance). Best when base model is high-bias (shallow trees = weak learners). XGBoost/LightGBM typically outperform RF on structured tabular data. However, boosting can overfit with too many rounds on noisy data — needs careful tuning of learning rate and regularization. Rule of thumb: RF first (robust, less tuning), XGBoost when you need to squeeze out more performance.

Gradient Boosting & XGBoost

Critical Depth Probe

MNC interviewers specifically ask: "Why does XGBoost use second-order gradients?" and "What's the difference between GBDT and Random Forest?" Know the math, not just the name.

Gradient Boosting idea: Build an additive model F(x) = Σ f_t(x). Each new tree f_t fits the negative gradient of the loss function at the current ensemble's predictions — i.e., the residuals (for MSE loss). Each tree corrects what the previous ensemble got wrong.

F_t(x) = F_{t-1}(x) + η·f_t(x) where f_t fits -∂L/∂F_{t-1}

Why "gradient" boosting? The residuals in MSE are exactly the negative gradient of MSE w.r.t. predictions. Gradient boosting generalizes this to any differentiable loss function — fit the negative gradient regardless of what the loss is.

XGBoost key innovation — second-order gradients (Taylor expansion): Instead of using only the first-order gradient (like vanilla GBDT), XGBoost uses a second-order Taylor approximation of the loss:

L ≈ Σ [g_i·f(x_i) + (1/2)·h_i·f(x_i)²] + Ω(f)

where g_i = ∂L/∂ŷ_i (first-order gradient) and h_i = ∂²L/∂ŷ_i² (second-order/Hessian). This gives a more accurate approximation → can take larger steps → converges faster. Also allows closed-form solution for optimal leaf weights.

Regularization in XGBoost: Ω(f) = γ·T + (λ/2)·‖w‖² where T = number of leaves, w = leaf weights. Penalizes tree complexity directly in the objective — this is why XGBoost rarely overfits with proper tuning.

Key hyperparameters: n_estimators (trees), learning_rate (η, shrinks each tree's contribution), max_depth, subsample (row sampling per tree), colsample_bytree (feature sampling), lambda (L2), alpha (L1).

The Interrogation

Killer Why does XGBoost use second-order gradients and what advantage does that give? ▼

Vanilla GBDT uses first-order gradient (steepest descent direction) which is locally linear. XGBoost uses a second-order Taylor approximation which includes curvature information. Benefits: (1) More accurate step: First-order only knows "go this direction." Second-order also knows "how sharply curved is the loss here?" → can take an optimally-sized step (like Newton's method). (2) Closed-form optimal leaf weights: With the second-order approximation, optimal weight for each leaf is w* = -G_j/(H_j + λ) where G_j = sum of first-order gradients in leaf j, H_j = sum of Hessians. No line search needed. (3) Hardware efficiency: The approximation allows pre-computing g_i and h_i, then solving the tree structure problem efficiently. (4) Better handling of non-MSE losses: First-order assumes loss is linear in leaf outputs. Second-order handles arbitrary convex losses more accurately.

SVMs & The Kernel Trick

Core idea: Find the hyperplane that separates classes with the maximum margin. The margin is the distance between the hyperplane and the closest points of each class (support vectors). Maximum margin → best generalization.

Hard margin SVM (linearly separable data): Maximize 2/‖w‖ subject to y_i(wᵀx_i + b) ≥ 1. Equivalent to minimizing ‖w‖² (quadratic programming problem). Only possible when data is perfectly separable.

Soft margin SVM: Allows some misclassification via slack variables ξ_i ≥ 0. Objective: min ‖w‖²/2 + C·Σξ_i. C controls tradeoff: large C → penalize misclassification heavily (low bias, high variance). Small C → allow more violations (high bias, low variance).

Kernel trick: SVM only uses dot products between data points (via the dual formulation). The kernel function K(x_i, x_j) = φ(x_i)ᵀφ(x_j) computes the dot product in a high-dimensional feature space without explicitly computing φ(x). This allows non-linear decision boundaries at linear computational cost.

Common kernels: Linear: K(x,z) = xᵀz. Polynomial: K(x,z) = (xᵀz + c)^d. RBF/Gaussian: K(x,z) = exp(-γ‖x-z‖²). RBF maps to infinite-dimensional space — can fit any function given enough data (infinite capacity).

SVM vs Neural Nets (2024): SVMs dominated 2000s; now largely replaced by deep learning for high-dimensional data. SVMs still useful for: small datasets, clear margin structure, interpretability, when you need a convex optimization guarantee.

The Interrogation

Depth Probe Why does the kernel trick work without computing the feature map? ▼

In SVM's dual form, the optimization and prediction only require dot products between training examples: K(x_i, x_j) = φ(x_i)ᵀφ(x_j). The Mercer kernel computes this value directly without computing φ(x_i) or φ(x_j) separately. For the RBF kernel: K(x,z) = exp(-γ‖x-z‖²). This corresponds to an infinite-dimensional feature space φ(x), but we never compute that vector — we just evaluate the kernel function on the original inputs. Cost: O(n·d) per kernel evaluation, not O(∞) for the feature map. The key insight: the Representer Theorem guarantees the optimal solution lies in the span of kernel evaluations at training points, so we never need the explicit feature map.

Clustering (k-means, DBSCAN)

Amazon Scenario Question (2025)

"What do you do when you don't know the number of clusters?" — Amazon directly asked this in a scenario-based ML round. Know DBSCAN, elbow method, and silhouette score.

k-means algorithm: (1) Initialize k centroids (randomly or k-means++). (2) Assign each point to nearest centroid. (3) Recompute centroids as mean of assigned points. (4) Repeat until convergence. Minimizes: Σ Σ ‖x - μ_k‖². Converges to local minimum — sensitive to initialization. k-means++ initialization reduces this problem.

Convergence guarantee: k-means always converges (objective is non-increasing each step) but to a local, not global minimum. Inertia (within-cluster sum of squares) is the objective.

When k is unknown: (1) Elbow method: Plot inertia vs k. Find the "elbow" — diminishing returns point. Subjective. (2) Silhouette score: For each point, s = (b-a)/max(a,b) where a = mean distance to same-cluster points, b = mean distance to nearest other-cluster points. Range [-1, 1]. s→1 = well-clustered. Pick k that maximizes average silhouette. (3) Gap statistic: Compares inertia to a random null distribution. More principled than elbow.

DBSCAN: Density-Based Spatial Clustering of Applications with Noise. No need to specify k. Parameters: ε (neighborhood radius), minPts (min neighbors to be core point). Core point: has ≥ minPts neighbors within ε. Clusters = connected components of core points + their neighbors. Points not reachable from any core point → noise. Handles arbitrary shapes, robust to outliers.

k-means vs DBSCAN: k-means: spherical clusters, requires k, sensitive to outliers. DBSCAN: arbitrary shapes, automatic k, identifies outliers, but sensitive to ε and minPts choice. For high-dimensional data: both suffer from curse of dimensionality.

The Interrogation

Mid Your data has clusters of different shapes and you don't know k. Which algorithm and why? ▼

Use DBSCAN. Reasons: (1) k-means assumes convex/spherical clusters — it will force circular boundaries even on crescent or ring-shaped data. (2) k-means requires specifying k up front — if unknown, you'd need to run elbow/silhouette analysis across many values of k, which is expensive. (3) DBSCAN discovers clusters of arbitrary shapes by following density gradients. (4) DBSCAN automatically labels outliers as noise — useful if data has anomalies. Caveat: DBSCAN needs ε and minPts tuning. If clusters have very different densities, HDBSCAN (hierarchical DBSCAN) handles this better by varying ε adaptively.

Regularization (L1, L2, Elastic Net)

Amazon Asked This (2025)

"What's the difference between L1 and L2 regularization? Why does L1 give sparse weights but L2 doesn't?" This was tested hands-on in Amazon's 2025 ML Deep Dive round. Know the geometric AND probabilistic explanation.

Why regularize? Without regularization, the model fits training data too closely (memorizes noise = overfitting). Regularization adds a penalty for complexity, trading some training accuracy for better generalization.

L2 (Ridge): Loss + λ·Σw_j². Penalizes large weights. Gradient: -λ·2w_j → shrinks weights proportionally toward zero but never exactly to zero (derivative never zero unless w=0). Solution: θ* = (XᵀX + λI)⁻¹Xᵀy. Probabilistic interpretation: Gaussian prior on weights.

L1 (Lasso): Loss + λ·Σ|w_j|. Gradient: -λ·sign(w_j). When w_j is small, the constant gradient magnitude λ·sign(w_j) can push it all the way to exactly 0. Result: sparse weights — feature selection built in. Probabilistic interpretation: Laplace prior on weights.

Geometric explanation for sparsity: Optimization = minimize loss subject to constraint on weight norm. L1 constraint = diamond shape (corners on axes). L2 constraint = sphere. The loss function's contours are ellipses centered at the unconstrained minimum. The first point where ellipses touch the L1 diamond is usually a corner (on an axis) → one or more weights = 0. The L2 sphere has no corners → the ellipses touch it on a curved surface → weights shrink but don't zero.

Elastic Net: λ₁·Σ|w| + λ₂·Σw². Combines sparsity (L1) with stability under correlated features (L2). Preferred when features are highly correlated (L1 arbitrarily picks one; Elastic Net can keep groups).

The Interrogation

Killer Explain mathematically why L1 drives weights to exactly zero but L2 only shrinks them. ▼

Consider a single weight w with loss L(w) + λ·|w| (L1) vs L(w) + λ·w² (L2).

L2: ∂/∂w [L(w) + λw²] = ∂L/∂w + 2λw = 0 → solution moves w toward 0 proportionally. For w to be exactly 0, we need ∂L/∂w = 0 at w=0 (unlikely unless L itself has minimum at 0).

L1: ∂/∂w [L(w) + λ|w|] is subgradient = ∂L/∂w + λ·sign(w). When w > 0: ∂L/∂w + λ = 0 → needs ∂L/∂w = -λ to stay non-zero. When w < 0: ∂L/∂w - λ = 0 → needs ∂L/∂w = +λ. If |∂L/∂w| < λ (i.e., the data doesn't provide enough gradient signal for this feature), the optimal solution is w = 0. The subgradient at w=0 includes the interval [-λ, λ], so 0 is a valid minimum whenever ∂L/∂w ∈ [-λ, λ]. This is why unimportant features (whose loss gradient is small) get exactly zeroed by L1 but only shrunk by L2.

Model Evaluation & Bias-Variance

Bias-Variance Decomposition: Expected test error = Bias² + Variance + Irreducible noise. Bias: error from wrong assumptions (underfitting — model too simple). Variance: error from sensitivity to training data fluctuations (overfitting — model too complex). Can't simultaneously minimize both — the tradeoff. Regularization, ensemble methods, and proper model complexity selection manage this tradeoff.

E[(y - ŷ)²] = [E[ŷ] - y_true]² + E[(ŷ - E[ŷ])²] + σ²_noise

Precision vs Recall: Precision = TP/(TP+FP) — of all predicted positives, how many are correct? Recall = TP/(TP+FN) — of all actual positives, how many did we catch? F1 = 2·(P·R)/(P+R) harmonic mean. Use precision when false positives are costly (spam filter: don't block legitimate email). Use recall when false negatives are costly (cancer detection: don't miss cancer).

AUC-ROC: Area Under the Receiver Operating Characteristic curve (TPR vs FPR at all thresholds). AUC = 0.5: random classifier. AUC = 1.0: perfect. AUC = probability that model ranks a random positive higher than a random negative. Insensitive to class imbalance — use AUC-PR (precision-recall) when imbalance is severe.

Cross-Validation: k-fold: split data into k folds, train on k-1, validate on 1, rotate. Average over k folds. Higher k = lower bias (more data used for training) but higher variance and computational cost. Leave-one-out (k=n): lowest bias, highest variance, expensive. For small datasets: use k=5 or k=10. Time-series: always use time-based split (never shuffle — data leakage).

The Interrogation

Depth Probe Your model has high training accuracy but poor test accuracy. Walk me through your diagnosis. ▼

This is high variance / overfitting. Systematic diagnosis: (1) Check the gap: How large is train vs validation error? Large gap → overfitting. (2) Learning curves: Plot train/val error vs training size. If they don't converge, more data could help. (3) Model complexity: Reduce — lower max_depth, fewer features, smaller model. (4) Regularization: Add L1/L2, dropout (NNs), early stopping. (5) Data: More training data is the best overfitting fix — variance decreases as n increases. (6) Feature engineering: Remove irrelevant features that the model is memorizing noise from. (7) Ensemble: Bag multiple models to average out variance. Rule: if model has too many parameters relative to data → high variance → reduce capacity or add more data.

Depth Probe When is AUC-ROC misleading and what do you use instead? ▼

AUC-ROC is misleading under severe class imbalance. Example: 99% negative, 1% positive. A model that scores all negatives slightly higher than positives gets high AUC but is useless (never detects positives at any useful threshold). Why: ROC uses FPR = FP/(FP+TN). With many negatives, FPR denominator is huge → FPR stays small even with many FPs → ROC curve looks good. Fix: Use AUC-PR (Precision-Recall curve). PR directly measures performance on the minority (positive) class. Under imbalance, a random classifier has AUC-PR ≈ prevalence (e.g., 0.01), so improvements are meaningful relative to this baseline. Also useful: F1 at the operating threshold, and Matthews Correlation Coefficient (MCC) which accounts for all four confusion matrix cells.
